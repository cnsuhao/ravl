Background

These classes provide a framework for the implementation of Bayesian reasoning systems. The background for this package can be found in Artificial Intelligence: A modern Approach, Chapter 14.

Basic concepts

The most basic is VariableC which represents random variables of 3 flavours: continuous, boolean and discrete. Next there is VariablePropositionC which represents a proposition that a variable takes a particular value, again in three flavours. E.g. we could have BooleanVariable=<true,false>. Then we can have a proposition BooleanVariable=true.

Next there is a VariableSetC which is pretty much just what it sounds like: it might be used for instance to describe the set of evidence variables. There is a corresponding PropositionC (????) class. Although this is related to a VariableSetC, it is not necessary for there to be a proposition for each variable in the set. However, you can't have a proposition for a variable not in the set, that is used to describe things like observed events. So for instance you could have a VariableSetC=(MaryCalls,JohnCalls,Alarm), and you could have a corresponding PropositionSetC=(MaryCalls=true,JohnCalls=false). Note that the proposition set says nothing about Alarm.

So typically we want to evaluate something like P(Burglary|MaryCalls,JohnCalls,Alarm). That query would have to return a full joint distribution. But to evaluate it you would need some numbers somewhere. You could alternatively request P(Burglary|MaryCalls=true,JohnCalls=false) which might return e.g. <0.284,0.716>, i.e. the values for Burglary=true and Burglary=false. These would be marginalised over Alarm. Note the combination of variable sets and proposition sets. Evaluating the P stuff involves quite a bit more infrastructure.

PDFs and CPDs

Next we need to consider PDFs and CPDs (conditional probability density functions). PDFs represent priors like P(Burglary) and P(Alarm)..... OK, so a PDF is a function of a single variable and can be evaluated for any particular proposition of that variable, so using the PDF you could get the prior probability P(Burglary=true). Of course you need a few concrete PDF implementations to handle the different types of variable, so there are boolean, discrete and continuous PDF classes. The boolean and discrete just use tables; the continuous would need functions. Currently there are uniform and normal continuous distributions, but more can be added.

We also need conditional probability distributions (CPDs) Essentially they are used to encode the table of values for e.g. P(Burglary|MaryCalls,JohnCalls,Alarm). A CPD has an output variable Burglary and a VariableSetC of parent variables Things get a bit hairy when you have continuous variables anywhere, but it is doable. There a few variants of CPDs implemented currently, so they can work with a few combinations for now. More combinations can be added later.

Bayesian Networks

Given this framework of PDFs and CPDs and variables and propositions we are in a position to define a Bayesian Network. More details are needed, but a network is defined in terms of a set of PDFs and CPDs, then you can ask it to evaluate different probabilities etc and it will internally do all the marginalisation etc. There is a BayesianNetworkC interface as well as a "simple" grid implementation BayesianNetworkSimpleC that exhaustively computes the probabilities rather than finding a statistical estimation. This doesn't scale well with variables so we would need to add a new concrete implementation based on a Markov Chain Monte Carlo (MCMC) method. However, the framework is pretty much all in place.
