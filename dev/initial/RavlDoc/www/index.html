What is RAVL ?

RAVL, Recognition And Vision Library, provides a base C++ class library together with a range of computer vision, pattern recognition and supporting tools.

The aim of RAVL is to move software developed within the Centre for Vision, Speech and Signal Processing, CVSSP at the University of Surrey, England for research purposes into the public domain and to support its use in a wider community.

RAVL is a public version of a library used internally within CVSSP called AMMA, which contains over 700,000 lines of C++ code. Much of RAVL's design, is based on this work. It is our intention to port as much of the AMMA code as possible into RAVL, removing code not compatible with the Lesser GNU Public License in the process.

FEATURES

Some of the features that set RAVL apart from other C++ libraries are:

  • SMP/thread-safe reference counting, allowing easy construction of large programs that takes full advantage of multiprocessor servers.
  • Powerful IO mechanism, allowing issues for file formats and type conversion to be handled separately from the main code.
  • JAVA-like class interfaces which largely avoid the direct use of pointers, allowing code to be written in a clear, readable style.
  • Easy-to-use and powerful make system suitable for building both large and small projects.

SUPPORTED PLATFORMS

RAVL is intended to work on a wide range of platforms and compilers. This is especially true of the `Core' module. The `OS' module, by definition, contains operating system specific code. The current implementation is oriented towards working on UN*X based platforms. The MSVC++ port of this code does compile but many of the interfaces are not yet implemented. It is intended that the MSVC++ port should be fully functional before too long.

Operating SystemProcessorCompiler
Linux i386 GNU, g++ (2.95.3 recommended)
Solaris Sparc GNU, g++ (2.95.x recommended)
IRIX Mips MIPS Pro
Windows NT i386 MSVC++, with latest service pack and CCFix
To compile RAVL Windows users will need the latest service pack for Visual C++, and a patch to the registry that allows it to understand '.cc' and '.hh' files. There's a utility which will do this fix automatically called CCFix .
