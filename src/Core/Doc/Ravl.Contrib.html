A "contrib" wrapper package usually contains classes that are essentially wrappers around non-RAVL libraries, for example for image I/O formats and hardware. Typically it provides a simplified and standardised interface to a software package with an incompatible (i.e. non-LGPL) licence.

The contrib code is included within RAVL in spite of the dependencies on these other libraries. However when building RAVL the inclusion of this code can be turned on or off. The documentation of any included contrib is merged into the overall RAVL tree. Some of these libraries are listed below:
