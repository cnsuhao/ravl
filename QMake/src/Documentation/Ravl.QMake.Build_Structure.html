QMake builds binary's and documentation into your PROJECT_OUT directory. This page describes its structure. The structure is based on the linux standard file system.

$PROJECT_OUT/bin Receives scripts and compiled programs. Compiled programs are implemented via a symbolic link to the most recently built binary for a particular program. If the last build was a qm opt, the binaries linked to in the bin directory will be the optimised versions. If the next build undertaken is a qm check, the links will be replaced with links to the the check binaries. The actual binary files will reside in the relevant sub-dir. (see below). This directory should appear on your PATH executable searchlist.
$PROJECT_OUT/bin/utils/$VAR{/shared{/single.X{.Y}}}/bin Receives compiled programs for the specific build type (VAR=opt, debug, check, etc. Optional shared directory level for programs linked against shared libraries. Futher optional directory level for binaries linked against particular single shared libraries - the .X .Y suffixes indicate the single libraries used to build the binary, appending the name of each single library that has been used).
$PROJECT_OUT/include/$PROJECT{/$PACKAGE} Receives the header files for the project (with an optional package level sub-directory).
$PROJECT_OUT/lib/$ARC/$VAR{/shared{/single}}/ Receives compiled libraries for the relevant build type (VAR=opt, debug, check, etc. Optional extra shared sub-dir for programs linked against shared libraries and a further extra single directory level for single shared libraries (SINGLESO)). Unlike executables, there is only ever one single directory for all of the single libraries. ).
$PROJECT_OUT/lib/$ARC/obj Receives object files needed for building static binaries
$PROJECT_OUT/lib/libdep Library dependancy files
$PROJECT_OUT/lib/libdep/$PLIB.def Library dependancy file for library PLIB
$PROJECT_OUT/lib/pkgconf Receives any pkgconfig meta file for the project
$PROJECT_OUT/QMake Receives a copy of QMake if one is being built for installation
$PROJECT_OUT/share/doc Receives the project documentation produced by the CxxDoc Documentation System
$PROJECT_OUT/share/QMake Receives a working copy of QMake with which to build Ravl if there is not currently a version of QMake installed on the system.
$PROJECT_OUT/test/$VAR{/shared{/single.X{.Y}}}/bin Receives compiled test programs for the relevant build type (VAR=opt, debug, check, etc. Optional extra shared sub-dir for programs linked against shared libraries, optional further layer for tests linked against specific single libraries).
$PROJECT_OUT/test/$VAR{/shared{/single.X{.Y}}}/log Receives the output log files from the relevant test programs. As usual, VAR=opt, debug, check, etc. the optional extra shared and single.X{.Y} sub-directories are for test programs linked against shared and single-shared libraries.
$PROJECT_OUT/transient Receives transient files from the build process. These files may be deleted once the project has been fully built. However, removal of these files will incur extra work should a subsequent build be required.
